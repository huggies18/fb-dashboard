## System Architecture Requirements (2026-04-21 05:57 UTC)

### User's Requirement: Senior Backend + Automation Engineer

User wants proper Queue + Worker architecture for Facebook Group Scraper:

#### Architecture:
1. **Queue** - กลุ่มรอ scrape [group_1, group_2, group_3, thaisolarcell]
2. **Worker** - รับ 1 group จาก queue → เปิด browser → scrape → save → notify → ปิด browser
3. **1 job = 1 group** (ห้าม loop หลาย group ใน browser เดียว)
4. **Browser reuse limit**: 2-3 groups ต่อ browser instance

#### /fbuyer Logic:
- scan post in group
- filter buyer keywords
- output: {post_id, text, author, timestamp, group_id}

#### /wbuyer Logic:
- ดึงข้อมูลจาก pending/DB
- export CSV: post_id, text, author, timestamp, group_id
- รันได้แม้ scraper ยังทำงาน

#### Anti-Detection:
- delay random 2-5 วินาที
- scroll แบบสุ่ม
- rotate proxy (ต่อ worker)
- user-agent realistic
- ห้าม request pattern เดิม

#### Error Handling:
- retry group ได้ 2 ครั้ง
- ถ้า error: kill browser, เปิดใหม่
- ห้าม retry infinite

#### Performance:
- worker 3-5 ตัว parallel (ไม่ share session)
- scale ได้
